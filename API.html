Clases para Modificar HTML

Estas clases son responsables de la manipulación del DOM, creación de formularios, tablas y elementos de la interfaz de usuario.

Dom

Clase principal que maneja la manipulación del DOM y la interfaz de usuario. Esta clase es responsable de crear y gestionar formularios, tablas, modales y otros elementos de la interfaz.

constructor()

Inicializa la clase Dom con las referencias básicas al formulario y configuración de columnas para tests. Establece las columnas disponibles y visibles por defecto para la visualización de resultados de tests.

createForm(action, estructura)

Crea un formulario dinámico basado en la estructura proporcionada y la acción solicitada. El método maneja diferentes tipos de formularios (ADD, EDIT, SEARCH, DELETE, SHOWCURRENT) y aplica lógica específica para cada tipo. Limpia el formulario existente y crea nuevos campos según la estructura de la entidad.
Parámetros:
  • action (string): Acción a realizar (ADD, EDIT, SEARCH, DELETE)
  • estructura (object): Estructura de la entidad

handleAddField(nombreAtributo, def)

Crea un campo específico para formularios de adición. Excluye campos autoincrementales y campos de archivo.
Parámetros:
  • nombreAtributo (string): Nombre del atributo a crear
  • def (object): Definición del campo
Retorna: HTMLElement - El elemento del campo creado o null si debe ser excluido

handleEditField(nombreAtributo, def)

Crea un campo específico para formularios de edición. Marca como readonly los campos que son clave primaria o campos de archivo.
Parámetros:
  • nombreAtributo (string): Nombre del atributo a crear
  • def (object): Definición del campo
Retorna: HTMLElement - El elemento del campo creado

handleSearchField(nombreAtributo, def)

Crea un campo específico para formularios de búsqueda. Marca como readonly los campos que son clave primaria o campos de archivo.
Parámetros:
  • nombreAtributo (string): Nombre del atributo a crear
  • def (object): Definición del campo
Retorna: HTMLElement - El elemento del campo creado

handleDeleteField(nombreAtributo, def)

Crea un campo específico para formularios de eliminación. Marca como readonly los campos que son clave primaria o campos de archivo.
Parámetros:
  • nombreAtributo (string): Nombre del atributo a crear
  • def (object): Definición del campo
Retorna: HTMLElement - El elemento del campo creado

createField(nombreAtributo, def, isReadOnly)

Crea un campo genérico para formularios. Maneja diferentes tipos de campos (input, select, etc.) y crea los elementos de error correspondientes.
Parámetros:
  • nombreAtributo (string): Nombre del atributo a crear
  • def (object): Definición del campo
Retorna: HTMLElement - El elemento del campo creado

colocarboton(action)

Crea un botón para el formulario. El botón se crea en el contenedor de botones del formulario.
Parámetros:
  • action (string): Acción a realizar (ADD, EDIT, SEARCH, DELETE, SHOWCURRENT)
Retorna: void - No retorna nada

load_data(contenido)

Carga los datos del formulario. Carga los datos del formulario.
Parámetros:
  • contenido (object): Los datos a cargar en el formulario
Retorna: void - No retorna nada

construirSelect()

Construye un select con las columnas disponibles para la tabla.
Retorna: void - No retorna nada

hacertabla()

Construye una tabla dinámica con los datos proporcionados. Maneja la creación de encabezados, filas y celdas, incluyendo la sanitización de datos para prevenir XSS. También agrega botones de acción para cada fila.
Retorna: void - No retorna nada

modificarcolumnasamostrar(atributo)

Modifica la visibilidad de las columnas en la tabla. Permite añadir o eliminar columnas de la visualización actual.
Parámetros:
  • atributo (string): Nombre del atributo a modificar
Retorna: void - No retorna nada

mostrarocultarcolumnas()

Muestra o oculta las columnas en la tabla.
Retorna: void - No retorna nada

mostrar_error_campo(id, codigoerror)

Muestra un mensaje de error para un campo específico. Actualiza la apariencia visual del campo y muestra el mensaje de error.
Parámetros:
  • id (string): ID del campo
  • codigoerror (string): Código o mensaje de error
Retorna: void - No retorna nada

mostrar_exito_campo(id)

Muestra un mensaje de éxito para un campo específico. Actualiza la apariencia visual del campo y muestra el mensaje de éxito.
Parámetros:
  • id (string): ID del campo
Retorna: void - No retorna nada

crearboton(entidad, accion, parametros)

Crea un botón para el formulario. El botón se crea en el contenedor de botones del formulario.
Parámetros:
  • entidad (string): La entidad a la que pertenece el botón
  • accion (string): La acción a realizar (ADD, EDIT, SEARCH, DELETE, SHOWCURRENT)
  • parametros (object): Los parámetros a pasar al botón
Retorna: void - No retorna nada

cerrar_formulario()

Cierra el formulario actual.
Retorna: void - No retorna nada

cerrar_test()

Cierra el test actual.
Retorna: void - No retorna nada

cerrar_tabla()

Retorna: void - No retorna nada

ocultar_boton_test()

Oculta el botón de test.
Retorna: void - No retorna nada

mostrar_boton_test()

Muestra el botón de test.
Retorna: void - No retorna nada

abrirModalError(errorMsg)

Abre un modal para mostrar mensajes de error.
Parámetros:
  • errorMsg (string): Mensaje de error a mostrar
Retorna: void - No retorna nada

cerrarModalError()

Cierra el modal de error y su overlay asociado.

mostrarModal()

Retorna: void - No retorna nada

Entidad_Abstracta

Clase abstracta base para todas las entidades del sistema. Esta clase proporciona la funcionalidad base para la gestión de entidades, incluyendo:

  • Gestión de formularios (creación, carga, validación)
  • Operaciones CRUD (ADD, EDIT, DELETE, SEARCH)
  • Manejo de tablas de datos
  • Integración con validaciones y tests
  • Gestión de archivos y datos especiales

constructor(entidad, def_estructura)

Inicializa la entidad abstracta con sus componentes básicos. Configura las validaciones, acceso externo y tests, además de inicializar la visualización del workspace.
Parámetros:
  • entidad (string): Nombre de la entidad
  • def_estructura (object): Definición de la estructura de la entidad

createFormGenerico(action, parametros = null)

Crea un formulario genérico para una acción específica. Proceso:
  1. Carga el formulario
  2. Carga los datos si se proporcionan
  3. Configura las validaciones
  4. Añade el botón de acción
  5. Configura los eventos del formulario
  6. Aplica la visibilidad de columnas según configuración
Parámetros:
  • action (string): Acción a realizar (ADD, EDIT, DELETE, SEARCH)
  • parametros (object): Parámetros opcionales para cargar en el formulario
Retorna: void - No retorna nada

change_value_IU(atributo, valor)

Método para transformar valores antes de mostrarlos en la interfaz. Puede ser sobrescrito por las clases hijas para implementar transformaciones específicas.
Parámetros:
  • atributo (string): Nombre del atributo
  • valor (any): Valor a transformar
Retorna: any - Valor transformado (por defecto retorna el valor original)

cargar_formulario(action)

Carga el formulario para una acción específica. Intenta usar un método estático si existe, o crea un formulario dinámico.
Parámetros:
  • action (string): Acción a realizar (ADD, EDIT, DELETE, SEARCH)
Retorna: void - No retorna nada

crearTablaDatos()

Crea y configura la tabla de datos. Proceso:
  1. Muestra la tabla
  2. Construye la estructura de la tabla
  3. Configura el selector de columnas
  4. Aplica la visibilidad de columnas según configuración
Retorna: void - No retorna nada

ADD()

Realiza la operación de añadir un nuevo registro. Proceso:
  1. Envía la solicitud al backend
  2. Maneja la respuesta
  3. Actualiza la interfaz según el resultado
Retorna: Promise - Promesa que se resuelve cuando la operación se completa

EDIT()

Realiza la operación de editar un registro existente. Proceso:
  1. Envía la solicitud al backend
  2. Maneja la respuesta
  3. Actualiza la interfaz según el resultado
Retorna: Promise - Promesa que se resuelve cuando la operación se completa

DELETE()

Realiza la operación de eliminar un registro. Proceso:
  1. Envía la solicitud al backend
  2. Maneja la respuesta
  3. Actualiza la interfaz según el resultado
Retorna: Promise - Promesa que se resuelve cuando la operación se completa

SEARCH()

Realiza una búsqueda en la entidad y actualiza la visualización de los resultados. Proceso:
  1. Envía la solicitud al backend
  2. Maneja la respuesta
  3. Actualiza la interfaz según el resultado
Retorna: Promise - Promesa que se resuelve cuando la búsqueda se completa

Clases para Validaciones

Estas clases manejan la validación de formularios, campos y datos de entrada del sistema.

Dom_validations

Clase que maneja las validaciones del DOM, extendiendo la funcionalidad de la clase Dom. Esta clase implementa un sistema completo de validación de formularios que incluye:

  • Validaciones atómicas (tamaño mínimo/máximo, expresiones regulares)
  • Validaciones de archivos (tipo, tamaño, formato de nombre)
  • Validaciones especiales por campo
  • Validaciones específicas por acción (ADD, EDIT, SEARCH, DELETE)

constructor(validaciones, entidad, entidadObj)

Inicializa la clase de validaciones con las dependencias necesarias. Configura el contexto de validación para una entidad específica y asegura que el método comprobarCampo esté disponible globalmente.
Parámetros:
  • validaciones (object): Instancia de la clase de validaciones atómicas
  • entidad (string): Nombre de la entidad a validar
  • entidadObj (object): Objeto de la entidad a validar

getAttributes()

Obtiene la estructura de atributos definida para la entidad actual. Evalúa dinámicamente la estructura de la entidad para obtener sus atributos y reglas de validación.
Retorna: Object - Objeto con la definición de atributos de la entidad

load_validations(action)

Carga y configura las validaciones para todos los campos del formulario. Asigna eventos de validación apropiados según el tipo de campo: - 'onblur' para inputs normales - 'onchange' para otros elementos
Parámetros:
  • action (string): Acción a realizar (ADD, EDIT, SEARCH, DELETE)
Retorna: void - No retorna nada

comprobarCampo(campo, action)

Valida un campo específico del formulario según la acción indicada. Realiza validaciones atómicas, especiales y específicas por acción.

  1. Verificación de existencia de definiciones para el campo
  2. Validación especial genérica
  3. Manejo específico para campos de archivo
  4. Validaciones según la acción (SEARCH, ADD, EDIT)

Tipos de validaciones soportadas:

  • min_size: Tamaño mínimo
  • max_size: Tamaño máximo
  • reg_exp: Expresión regular
  • valid_date: Validación de fecha
Parámetros:
  • campo (string): ID del campo a validar
  • action (string): Acción a realizar (ADD, EDIT, SEARCH, DELETE)
Retorna: boolean|string - true si el campo es válido, mensaje de error en caso contrario

submit_test(action)

Realiza una validación completa de todos los campos del formulario. Proceso:
  1. Obtiene todos los atributos con reglas de validación
  2. Filtra los atributos que tienen reglas para la acción específica
  3. Valida cada campo
  4. Recopila y reporta errores
Parámetros:
  • action (string): Acción a realizar (ADD, EDIT, SEARCH, DELETE)
Retorna: boolean - true si todos los campos son válidos, false si hay errores

check_special_tests(atributo, value)

Verifica si un valor específico cumple con las reglas especiales de validación. Proceso:
  1. Obtiene la definición de validación especial para el atributo
  2. Valida el valor según las reglas especiales
Parámetros:
  • atributo (string): Nombre del atributo a validar
  • value (any): Valor a validar
Retorna: boolean|string - true si el valor es válido, mensaje de error en caso contrario

  • Validaciones específicas para archivos

constructor()

  2. Obtiene el valor del campo
  3. Compara el tamaño del valor con el parámetro
Parámetros:
  • id (string): ID del campo a validar
  • parametro (number): Tamaño mínimo permitido
Retorna: boolean - true si el tamaño es válido

min_size(id, parametro)

Parámetros:

max_size(campo, param)

Valida que el tamaño del campo sea menor o igual al parámetro especificado. Proceso:

Acceso al Backend

Estas clases manejan la comunicación y acceso a servicios del backend del sistema.

ExternalAccess

Clase que maneja las comunicaciones con el backend del sistema. Proporciona métodos para realizar peticiones HTTP al servidor, específicamente para operaciones CRUD y otras interacciones con el backend.

constructor()

Inicializa la clase ExternalAccess.

back_request(action, data)

Realiza una petición POST al backend del sistema. Proceso:
  1. Prepara los datos del formulario o crea un FormData vacío
  2. Añade el controlador y la acción
  3. Añade datos extra si se proporcionan
  4. Realiza la petición AJAX
Parámetros:
  • action (string): Acción a realizar
  • data (object): Datos a enviar
Retorna: Promise - Resultado de la petición

Fichero de traducciones

Fichero que maneja las traducciones de los textos del sistema. Proporciona métodos para obtener las traducciones de los textos del sistema.

Idioma

Módulo de gestión de idiomas del sistema. Proporciona funcionalidades para cambiar y mantener el idioma de la interfaz, incluyendo la gestión de cookies para persistencia del idioma seleccionado.

setLang(lang)

Establece el idioma de la interfaz y aplica las traducciones correspondientes.

setCookie(name, value, days)

Establece una cookie con el nombre, valor y días de duración.

getCookie(name)

Obtiene el valor de una cookie.

cambiarLang(lang)

Cambia el idioma de la interfaz y recarga la página.
